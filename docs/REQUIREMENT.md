## 1. 产品概述

**Holdly** 是一款面向个人用户的资产持有成本追踪 PWA 应用。

**核心价值**：把购入价格摊薄成每日成本，让每件物品的真实代价清晰可见。

**两种资产类型**：
- **买断型**：一次性购入，按持有天数计算每日成本（动态递减）
- **订阅型**：周期性付费，按订阅周期天数计算每日成本

**计划模块**：多人协作的收支预算管理，支持 accumulate（收支累加）和 snapshot（总额记录）两种模式。

**货币**：全局仅支持单一货币单位，界面不展示货币符号，仅显示数字。

## 2. 功能特性

### 2.1 资产管理

**资产列表**：展示所有资产卡片，支持按名称搜索、按类型（买断型/订阅型/已取消）筛选、按分类 pill 筛选、按标签弹出列表筛选。支持 7 种排序（最近添加、价格升降序、每日成本升降序、购入日期正倒序）。每张卡片显示 emoji、名称、状态指示点、每日成本、类型标签、分类、标签、价格、持有天数。

**资产详情（买断型）**：Hero 区展示 emoji + 名称 + 状态徽标（持有中/已卖出/已换新）。财务信息包括购入价、当前估价、每日成本（高亮）、持有天数、支付方式、购入日期。支持以下操作：
- **卖出资产**：弹出 Dialog，输入卖出价格和卖出日期
- **以旧换新**：跳转到换新页面，填写回收价、新设备信息
- **编辑保修**：弹出 Dialog，填写保修起止日期和备注（upsert，每个资产最多一条保修）
- **添加/编辑/删除维修记录**：通过底部 Sheet 操作，字段包括维修日期、费用、原因、维修商、结果、是否完成。删除无确认直接执行
- **删除资产**：二次确认后软删除

**以旧换新流程**：填写回收价和换新日期后，系统标记旧资产为已换新（不软删除），并创建新资产。新资产自动继承旧资产的 emoji 和分类，购入价自动计算为新设备标价减去回收价。系统自动为新资产打上「以旧换新购买」标签。换新测算面板展示新旧每日成本对比。

**订阅详情**：展示周期标签（月付/季付/年付）、订阅价、每日成本、下次续费日期、开始日期、持有天数。支持以下操作：
- **取消订阅**：弹出 Dialog，选择取消日期，设置状态为 cancelled
- **恢复订阅**：一键恢复为 active 状态
- **编辑**：跳转编辑页
- **删除**：二次确认后软删除

### 2.2 统计总览

**KPI 卡片（4 个）**：每日成本（所有持有中资产当日日费之和）、订阅费用（活跃订阅月度费用之和）、资产数量（活跃资产总数）、资产总额（活跃资产购入价之和）。

**分类花费分布**：水平进度条展示各分类的花费占比和金额，支持买断型/订阅型切换查看。

**月度趋势图**：折线面积图，展示过去 6 个月每月花费趋势，支持买断型/订阅型切换。

**即将到期**：展示未来 30 天内到期的订阅续费和保修到期项目。顶部有可关闭的警告横幅，点击可跳转至对应资产详情页。

### 2.3 协作计划

**计划列表**：展示用户参与的所有计划卡片，显示 emoji、名称、模式徽标（收支累加/总额记录）、成员头像、权限标签、最近净收入。

**计划详情**：净值趋势图（12 个月，支持净值/净收入切换）、KPI（最近净收入、当前总额）、月度记录列表（按月倒序）。Owner 可管理邀请链接和删除计划。

**月度记录**：查看页展示收入合计、支出合计、净收入 KPI，以及收入/支出明细（含成员头像、项目名、金额）和成员备注。删除记录需输入月份字符串确认。

**编辑月度记录**：年/月选择器切换月份。收入/支出条目支持增删改，每条包含成员归属、项目名、金额。own 权限模式下其他成员的条目半透明禁用不可编辑。成员备注可编辑。保存时使用乐观锁（校验更新时间戳防止冲突）和 Diff 保存（仅发送增量变更：addedItems、updatedItems、deletedItems、memberNotes）。

**邀请链接**：Owner 可生成 30 天有效期的随机 token 邀请链接，可撤销当前活跃链接。被邀请者通过链接加入计划后成为 editor 成员。

**CSV 导入**：snapshot 模式支持导入历史数据。CSV 格式：首列为记录日期（YYYY-MM-DD 或 YYYY/M/D），后续列为 `email|项目名` 格式的成员+项目列，末列可选备注。

**计划编辑**：可修改 emoji、名称、模式（accumulate/snapshot）、协作权限（own/all）、起始数字、预设收支项（增删改）。编辑页支持邀请链接管理和 CSV 历史导入。

### 2.4 设置

**个人资料**：emoji 头像（点击弹出 EmojiPicker 修改）、昵称（行内编辑，max 30）、邮箱（只读展示）。

**分类管理**：列表展示所有分类（含预置标记）。支持新建（emoji + 名称）、行内编辑名称、软删除。删除时关联资产的分类自动清除为 null。预置分类不可删除。

**标签管理**：列表展示所有标签（颜色点 + 名称 + 关联资产计数）。支持新建（颜色选择器 + 名称）、行内编辑名称和颜色、软删除。删除时清除所有 asset_tags 关联。

**支付类型管理**：列表展示所有支付类型。支持新建、行内编辑、软删除。预置类型不可删除。删除时级联软删除下级支付账户。

**支付账户管理**：列表展示所有账户（名称 + 支付类型徽标）。支持新建（选择支付类型 + 名称）、行内编辑、软删除。

**主题切换**：自动（跟随系统）/ 浅色 / 深色，即时切换无需保存。

**退出登录**：清除 session 后跳转到登录页。

### 2.5 邮件提醒

- 订阅续费到期提醒
- 保修到期提醒
- 支持全局默认和单资产覆盖两种提醒天数配置

### 2.6 PWA

- 可添加到主屏幕（iOS/Android）
- 离线缓存、自动更新

## 3. 业务规则

### 3.1 持有天数

```
买断型：n(D) = D - purchaseDate + 1（购买日当天 = 第 1 天）
买断已卖出：总持有天数 = soldDate - purchaseDate（卖出日不计入，同日取 1）
订阅型：active_days = 区间与 [startDate, endDate) 的交集天数
```

### 3.2 每日成本

**买断型**（动态递减）：

```
dailyCost = purchasePrice / holdingDays
```

持有天数越多，日均成本越低。

**订阅型**（固定）：

```
monthly  → price / 30
quarterly → price / 91
yearly   → price / 365
```

### 3.3 区间成本

买断型使用调和级数逐项求和，订阅型使用 `（price / 30）× activeDays`。详见开发架构文档。

### 3.4 以旧换新

```
新资产实际支出 = 新设备标价 - 回收价
旧资产净成本 = 购入价 - 回收价
```

换新后旧资产标记 `tradedInAt` + `tradeInPrice`（不软删除），新资产通过 `tradedFromAssetId` 关联旧资产。

### 3.5 续费日推算

优先使用数据库 `nextRenewalDate` 字段，若无则推算：

```
续费日 = subscriptionStartDate + k × 周期月数
// monthly=1, quarterly=3, yearly=12
到期日 = 续费日 - 1天
```

Dashboard 中展示的到期天数基于到期日计算。

### 3.6 软删除

所有业务数据使用软删除（`deleted_at`），禁止硬删除用户数据。唯一的例外是维修记录。

### 3.7 Plans 净值

```
accumulate 模式：月净收入 = Σ(收入) - Σ(支出)，总额 = 起始值 + Σ(所有月净收入)
snapshot 模式：总额 = 手动录入的 recordedTotalValue
```

### 3.8 邮件提醒天数

| 提醒类型 | 默认提前 | 可选值 |
|---|---|---|
| 订阅续费到期 | 7 天 | 3天 / 7天 |
| 保修到期 | 14 天 | 7天 / 14天 / 30天 |

优先级：单资产覆盖 > 全局默认。

## 4. 页面需求 — 认证

| 元素 | 说明 |
|---|---|
| GitHub OAuth 按钮 | 跳转 GitHub 授权 |

通过邮件重置链接进入，设置新密码。

## 7. 页面需求 — 计划模块

### 7.1 计划列表 `/plans`

展示用户参与的所有计划卡片：emoji + 名称 + 模式徽标（收支累加/总额记录）+ 成员头像 + 权限标签 + 最近净收入

### 7.2 新建/编辑计划

路由：`/plans/new`、`/plans/:id/edit`

| 字段 | 说明 |
|---|---|
| emoji | emoji 选择器 |
| 名称 | 必填 |
| 模式 | accumulate（收支累加）/ snapshot（总额记录）|
| 协作权限 | own（成员仅编辑自己）/ all（成员可编辑全部）|
| 起始数字 | 净值起始值，默认 0 |
| 默认子项 | 预设收支项名称，创建月记录时自动填充 |
| 成员 | 通过邀请链接加入，可设备注；owner 可移除成员 |
| CSV 导入 | snapshot 模式支持导入历史数据 |

编辑页支持邀请链接管理（生成/撤销）、成员移除（AlertDialog 二次确认）和 CSV 历史导入。

### 7.3 计划详情 `/plans/:id`

- 净值趋势图（支持三种模式切换，即时切换无服务端请求）：
  - **净值**：近 12 个月净值趋势
  - **净收入**：近 12 个月月净收入趋势
  - **年度**：近 6 年年度净收入趋势
- KPI：最近净收入、当前总额
- 月度记录列表（按月倒序）：支持顶部年份选择器（`<Select>`）按年份筛选。每月显示总额、净收入、收支明细预览。点击月份卡片进入记录详情，点击铅笔图标进入编辑
- 邀请链接管理（仅 owner）：生成/重新生成/撤销邀请链接
- 「添加记录」按钮：跳转到当月空记录编辑页
- 删除计划：`···` 菜单 → 弹出二次确认 → 输入确认后软删除，跳转 `/plans`

### 7.4 月度记录 `/plans/:id/records/:month`

- KPI：收入合计、支出合计、净收入
- 收入/支出明细：每条显示成员头像、项目名、金额，按类型分组
- 成员备注区：各成员的备注文本
- 编辑按钮：跳转到编辑页
- 删除记录：`···` 菜单 → 弹出 Dialog → 需手动输入月份字符串（如 `2026-05`）确认 → 不匹配则显示错误提示 → 匹配后软删除并跳转计划详情

### 7.5 编辑月度记录 `/plans/:id/records/:month/edit`

- 年/月选择器：顶部下拉选择年份和月份
- 收入条目：点击「添加收入项」添加空行，每行有项目名输入框和金额输入框，支持编辑和删除
- 支出条目：点击「添加支出项」添加空行，同上
- 权限控制：`own` 模式下，其他成员的条目半透明显示且不可编辑；`all` 模式下所有条目可编辑
- 成员备注：当前用户可编辑自己的备注
- 保存机制：
  - 乐观锁：保存时携带 `expectedRecordUpdatedAt`，服务端校验不一致则返回冲突错误
  - Diff 保存：仅发送增量变更（`addedItems` 新增条目、`updatedItems` 修改条目、`deletedItems` 删除条目 ID、`memberNotes` 备注更新）
  - 新建空记录支持：通过 `?blank=1` 参数创建无预填项的记录
- 保存成功后跳转到记录详情页

### 7.6 邀请链接

- 生成：30 天有效期随机 token
- 接受：通过邀请链接加入计划
- 撤销：owner 可撤销当前活跃链接

### 7.7 CSV 导入

snapshot 模式支持，格式要求：
- 首列为「记录日期」（YYYY-MM-DD 或 YYYY/M/D）
- 后续列为 `email|项目名` 格式的成员+项目列
- 末列可选「备注」

## 8. 页面需求 — 设置

### 8.1 设置主页 `/settings`

- 用户信息：emoji 头像（点击弹出 EmojiPicker 修改，提交即保存）+ 昵称（点击铅笔图标进入行内编辑模式，输入后点击对勾确认，× 取消，max 30 字符）+ 邮箱（只读展示）
- 外观：主题切换按钮组（自动/浅色/深色），即时切换，无需保存
- 数据管理入口：四个导航项，每项显示名称和关联数据计数（分类(7)、标签(0)、支付类型(7)、支付账户(0)）
- 退出登录：点击后清除 session，跳转 `/login`

### 8.2 分类管理 `/settings/categories`

- 列表：每个分类显示 emoji + 名称 + 关联资产计数，预置分类显示「预置」徽标
- 新建：emoji 选择器 + 名称输入 +「新增」按钮提交
- 编辑：点击铅笔图标进入行内编辑模式，可修改名称，点击对勾确认
- 删除：点击垃圾桶图标，软删除。关联资产的 `category_id` 自动设为 null。预置分类不可删除

### 8.3 标签管理 `/settings/tags`

- 列表：每个标签显示颜色圆点 + 名称 + 关联资产计数
- 新建：颜色选择器（默认 #cc785c）+ 名称输入 +「新增」按钮提交
- 编辑：点击铅笔图标进入行内编辑，可修改名称和颜色
- 删除：点击垃圾桶图标，软删除并清除所有 `asset_tags` 关联

### 8.4 支付类型管理 `/settings/payment-types`

- 列表：每个类型显示名称，预置类型不可删除
- 新建：名称输入 +「新增」按钮提交
- 编辑：点击铅笔图标进入行内编辑
- 删除：点击垃圾桶图标，软删除该类型并级联软删除其下所有支付账户。预置类型不可删除

### 8.5 支付账户管理 `/settings/payment-accounts`

- 列表：每个账户显示名称 + 所属支付类型徽标
- 新建：选择支付类型（下拉）+ 名称输入 +「新增」按钮提交
- 编辑：点击铅笔图标进入行内编辑，可修改名称
- 删除：点击垃圾桶图标，软删除

## 9. 未实现

以下功能在需求设计中存在，但当前代码中尚未实现或仅为 UI 占位。

### 9.1 提醒设置持久化

资产详情页和订阅详情页有提醒设置 Dialog，但「保存」按钮仅关闭 Dialog，无后端持久化。

### 9.2 设置页通知配置

设置主页的「邮件到期提醒」开关及天数配置被注释掉。

### 9.3 手动续费操作

订阅详情页无「续费」按钮。续费日期仅自动计算展示，无法手动记录续费支付。

### 9.4 移动端 FAB

移动端底部 Tab Bar 中间应有浮动操作按钮用于快速新建资产，当前未实现。

### 9.5 空状态 UI

资产列表页在无资产或筛选无结果时，无插图或引导文案。

### 9.6 购买凭证上传

资产表单中凭证字段为文本输入，不支持文件/图片上传。

### 9.7 CSV 导出

计划模块支持 CSV 导入，但无导出功能。

### 9.8 法律页面

注册页的「服务条款」和「隐私政策」链接指向 `#`，实际页面未创建。

### 9.9 支付账户按类型分组

支付账户管理页面为平铺列表，无按支付类型分组的分组标题。

### 9.10 分类 emoji 编辑

已有分类只能修改名称，无法修改 emoji 图标。

### 9.11 Plans 邀请页面未登录处理

邀请页面在用户未登录时重定向至登录页，但不保留邀请 token。
